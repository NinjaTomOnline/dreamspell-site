Data used in the current release

  • Dreamspell reference data is bundled inside the app and used locally on device.
  • Journal entries are stored locally on the device using Apple system persistence.
  • The app does not require an account to use the shipped feature set.
  • The current release docs do not describe third-party analytics, third-party advertising, or server-backed journal storage.

Permissions and future-facing features

  • The current shipping release does not include active contacts import.
  • The app contains usage-description copy for a future opt-in contacts update, but that feature is not active and should not be advertised as current behavior.
  • If a future version enables contact-based or other permission-gated features, this policy should be updated before that version is released.

Tracking, Sharing, And Retention

How the current release handles user data

This public policy keeps to the behavior already described in the repo’s submission-prep materials and avoids making promises about features that are not live yet.

Tracking and advertising

The current Dreamspell release is described as not using app data for tracking across apps or websites, not including third-party advertising, and not selling user data.

Data sharing

The current release is described as not transmitting journal content or Dreamspell calculation data to the developer, and not sharing journal entries or Dreamspell readings with third parties.

Data retention

Journal entries remain on the device until the user edits or deletes them, or removes the app and its local data. Bundled reference data remains part of the installed app package.
